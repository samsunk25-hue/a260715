# 🐉 스터디 드래곤 (Study Dragon)

> 나만의 학습 탐험일지 — 매일 미션을 완료하고 5단계로 진화하는 드래곤을 키우는 중학생용 학습 앱

**▶ 앱 주소: https://study-dragon.web.app**

## 이게 뭔가요

학습을 '지루한 의무'가 아닌 '성장 게임'으로 만드는 앱입니다.
타인과의 비교나 랭킹은 **의도적으로 넣지 않았습니다.** 경쟁이 아니라 스스로의 성장에 집중하도록요.

## 주요 기능

| 기능 | 설명 |
|---|---|
| 🎨 **다이내믹 컬러 미션** | 완료 칸 수에 따라 테마가 회색 → 파랑 → 초록 → 주황 → 핑크로 변화 |
| 🥚 **5단계 드래곤 진화** | 이번 달 하트에 따라 🥚 → 🐣 → 🦎 → 🐲 → 🐉, 진화하면 축하 팝업 |
| 🗓️ **매월 1일 리셋** | 드래곤이 알부터 다시 시작 (지난 기록은 그대로 보존) |
| 💕 **전체 응원** | 교사·학부모가 남긴 응원이 그날 접속하는 모든 학생에게 표시 |
| 📊 **7일 성장 그래프** | 캐릭터를 누르면 최근 7일 달성도를 막대그래프로 |
| 📔 **주간 회고 일기** | 일요일에만 열리는 자기점검 공간 |
| 🎉 **완벽 달성 팡파르** | 4칸을 다 채우면 하트가 ⭐로 바뀌고 꽃가루가 터짐 |
| 🛡️ **비속어 필터** | 응원과 일기의 욕설을 `●`로 가림 |
| 🔒 **당일만 체크** | 지난 날 몰아치기·앞날 미리 찍기 방지 (보기는 가능) |
| 📈 **교사용 익명 통계** | 🧒 버튼 안에서 반 전체 달성 현황 확인 |

## 드래곤 진화 기준

**이번 달** 하트 기준입니다. 매월 1일에 알로 돌아갑니다.

| 레벨 | | 이름 | 이번 달 하트 |
|---|---|---|---|
| Lv.1 | 🥚 | 신비한 알 | 0 ~ 9 |
| Lv.2 | 🐣 | 꼬마 해치 | 10 ~ 24 |
| Lv.3 | 🦎 | 날쌘 아기용 | 25 ~ 44 |
| Lv.4 | 🐲 | 용맹한 드래곤 | 45 ~ 69 |
| Lv.5 | 🐉 | 마스터 드래곤 | 70 ~ |

한 달 최대는 약 120하트(30일 × 4칸)입니다. Lv.5의 70은 하루 평균 2.3칸으로,
성실한 학생이 도달할 수 있는 수준입니다.

> ⚠️ **기준을 올리실 때 주의하세요.** 처음엔 Lv.5가 100이었는데, 그건 평생 누적
> 기준이라 가능했던 숫자입니다. 월간으로 바뀐 지금 100은 **25일을 완벽하게 채워야**
> 도달합니다. 그대로 뒀다면 마스터 드래곤을 아무도 못 봤을 겁니다.

**리셋 방식**: 지난 기록을 **지우지 않습니다.** 세는 범위만 이번 달로 좁힐 뿐입니다
([`app.js`](app.js)의 `monthHearts`). 그래서 1일이 되면 자연히 0이 되고, 캘린더의
지난 하트와 7일 그래프는 그대로 남습니다. 지울 게 없으니 리셋이 실패할 수도 없습니다.

## 교사용 익명 통계

🧒 버튼(전체 응원 작성) 안에 **"오늘의 우리 반"** 이 있습니다.
시작한 학생 수, 평균 달성, 완벽 달성 인원, 0~4칸 분포를 보여줍니다.

**개인은 절대 특정되지 않습니다.** 저장되는 건 `{ date, count }` 뿐이고,
기기 구분자는 앱이 만든 난수라 사람과 연결되지 않습니다. 보안 규칙의
`hasOnly(['date','count','updatedAt'])` 가 **이름 같은 필드를 넣으려는 시도를
데이터베이스 차원에서 거부**합니다 — 앱 코드에 버그가 생겨도 개인정보는 들어갈 수 없습니다.

> 이 화면을 학생 화면이 아니라 교사 화면에 둔 이유: PRD 2장이 "타인과의 비교나
> 랭킹 등 소셜 기능 완전 배제"라고 못박고 있습니다. 학생에게 "6명이 4칸 완료"를
> 보여주면 랭킹은 아니어도 비교가 됩니다.

## 비속어 필터

응원 메시지와 주간 일기에 욕설이 들어가면 `●`로 가립니다. 단어 목록은
[`badwords.js`](badwords.js) 맨 위에서 편집할 수 있습니다.

**동작 시점이 서로 다릅니다.**

| | 언제 거르나 | 왜 |
|---|---|---|
| 응원 | [메시지 띄우기] 누를 때 | 모든 학생에게 공개되는 글이라, 몰래 고쳐 보내지 않고 무엇이 가려졌는지 보여준 뒤 확인을 받음 |
| 일기 | 입력창을 벗어날 때 | 타이핑 중에 거르면 "시발점"을 쓸 수 없음 — "시발"까지 쳤을 때 `●●`가 되어버리기 때문 |

### 오탐 방지가 핵심입니다

한국어 욕설 필터에서 제일 흔한 실패는 **멀쩡한 말을 가려버리는 것**입니다.
`시발점`, `개발자`, `새끼손가락`, `미친듯이`, `병실` 같은 단어가 대표적이고,
[`badwords.js`](badwords.js)의 `SAFE` 목록이 이들을 보호합니다.

**공백은 일부러 지우지 않습니다.** 공백을 지우고 검사하면 아래처럼 됩니다:

```
"역시 발표를 잘했다"  →  "역● ●표를 잘했다"   (!!)
"도시 발전에 대해"    →  "도● ●전에 대해"     (!!)
"혹시 발견한 사람"    →  "혹● ●견한 사람"     (!!)
```

한국어는 `역시`/`도시`/`혹시`/`다시` 뒤에 `발`로 시작하는 말이 오는 경우가
흔해서, 실제로 평범한 문장 11개 중 7개가 걸렸습니다.

### 알려진 한계

- **`"시 발"`처럼 공백을 끼운 우회는 못 잡습니다.** `"역시 발표"`와 글자 순서가
  똑같아서 문맥 없이는 구분할 수 없습니다. 정상 문장을 지키는 쪽을 택했습니다.
- `"시이이발"`처럼 글자를 늘린 우회도 못 잡습니다.
- **필터는 브라우저에서 동작합니다.** 로그인이 없는 구조라 Firestore 주소를
  아는 사람은 필터를 거치지 않고 직접 응원을 넣을 수 있습니다. 교실에서 쓰는
  수준의 실수를 막는 장치이지, 작정한 공격을 막는 장치가 아닙니다.

## 로컬에서 실행하기

> ⚠️ **`index.html`을 더블클릭하면 안 됩니다.**
> `<script type="module">`은 `file://` 주소에서 브라우저 보안정책(CORS)에 막힙니다.
> 반드시 `http://` 주소로 열어야 합니다.

**방법 1 — 포함된 서버 스크립트 (설치 불필요)**

`서버실행.ps1` 우클릭 → **PowerShell에서 실행**
브라우저가 자동으로 `http://localhost:5500` 을 엽니다.

**방법 2 — VS Code Live Server**

`Live Server` 확장 설치 → `index.html` 우클릭 → **Open with Live Server**

## 데이터 설계

로그인이 없는 앱이라, 데이터를 이렇게 나눴습니다:

| 데이터 | 저장 위치 | 이유 |
|---|---|---|
| 미션 체크 / 누적 하트 / 드래곤 / 일기 | 브라우저 localStorage | 개인 것. 로그인이 없으니 누구 건지 구분할 방법도 없음 |
| **전체 응원 메시지** | **Firebase Firestore** | 교사·학부모 → 모든 학생. 공유가 필수 |

덕분에 Firestore에는 컬렉션 하나만 있으면 됩니다.

```
cheers/{자동ID}
  ├─ date: "2026-07-15"    ← 문서 ID가 아니라 필드
  ├─ message: "오늘도 힘내자!"
  └─ createdAt: 타임스탬프
```

날짜를 **필드**로 둔 덕에 범위 쿼리 한 번으로 그 달 전체를 가져와서,
캘린더의 핑크 뱃지와 모달의 응원 카드를 **조회 1번으로 함께** 해결합니다.

## Firebase 설정 (전체 응원 기능)

설정하지 않아도 **나머지 기능은 전부 정상 동작합니다.** 응원만 잠겨 있습니다.

1. [Firebase 콘솔](https://console.firebase.google.com)에서 프로젝트 생성
2. **Firestore Database** 생성 — 위치는 `asia-northeast3 (Seoul)`
3. **프로젝트 설정 → 웹 앱(`</>`) 추가** → `firebaseConfig` 복사
4. `firebase.js` 상단의 `firebaseConfig`에 붙여넣기
5. Firestore **규칙** 탭에 아래를 붙여넣고 게시

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /cheers/{id} {
      allow read: if true;
      allow create: if request.resource.data.message is string
                    && request.resource.data.message.size() > 0
                    && request.resource.data.message.size() < 200
                    && request.resource.data.date is string;
      allow update, delete: if false;   // 응원은 쌓이기만 하고 사라지지 않음
    }
  }
}
```

> `apiKey`는 비밀번호가 아닙니다. Firebase 웹 설정값은 원래 모든 사용자의 브라우저에
> 노출되도록 설계된 값이라 공개해도 됩니다. 실제 보안은 위 **규칙**이 담당합니다.

### 알려진 한계

로그인이 없어서 **주소를 아는 사람은 누구나 응원을 쓸 수 있습니다.**
교실 시연·과제 제출 수준에서는 괜찮지만, 실제로 여러 학교에 배포한다면
Firebase Authentication이 필요합니다. 부적절한 메시지는 Firebase 콘솔에서 직접 삭제해야 합니다.
